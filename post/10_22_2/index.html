迭代器

迭代器类似于指针，提供了对对象的间接访问，string类型和容器都支持迭代器的操作。和指针不同的是获取迭代器不是使用取地址符。典型的成员函数 begin返回指向的第一个元素的迭代器。end指向尾元素的下一个位置，该迭代器只是个标记表示已经处理完容器中的所有元素。特殊情况下如果容器为空，begin和end返回同一个迭代器。

auto b = v.begin; auto e = v.end;

迭代运算符及类型

和指针类似使用解迭代器来获取所指示的元素，执行解引用的迭代器必须合法并确实指示着某个元素。迭代器的类型：iterator ,const_iterator 。 如果对象为常量必须使用const修饰的迭代器。cbeing cend 函数返回值类型都是const_iterator

2

vector<int>::iterrator it; //it能读写元素
string::const_iterator it2; // it2只能读取元素不能写

某些对vector对象的操作可能会使迭代器失效.但凡是使用迭代器的循环体，都不要向迭代器所属的容器内添加元素。

迭代器运算

两个迭代器指向同一个容器的元素或者尾元素的下一个位置，就能相减得到两个元素的距离。得到的类型名为 difference_type 的带符号整型数。距离可以为正也可以为负。

迭代器运算的一个经典算法是二分法搜索

}

数组

数组是类似于vector的一种数据结构，存储同样类型的对象的容器，这些对象没有名字，需要通过其所在的位置访问。但是数组大小去确定不变，不能随意向数组中添加元素，如果不清楚数组的确切个数，使用vector.

定义和初始化内置数组

显式初始化数组元素：可以对数组进行列表初始化，此时可以忽略数组的维度，但是初始值不能大于数组维度。

字符数组可以用字符串字面值初始化。不能将数组的内容拷贝给其他数组作为其初始值，也不能为其他数组赋值。

指针也是迭代器

如同迭代器一样，遍历数组需要知道数组的首元素地址和尾后指针为此C+11提供了 begin和end函数，功能与上文的同名函数类似。两个函数定义在iterator头文件中。尾后指针不能进行解引用

C风格字符串

C风格字符串不是一种类型，而是为了表达和使用字符串而形成的一种约定俗成的写法，按此习惯性的写法的字符串在字符数组中以 空字符结束 一般利用指针来操作这些字符串。

  • 传入函数中的C风格字符串必须跟着’\0’以表示结束该字符串,否则字符串可能会在内存中不断寻找，直到遇到空字符才停下来。

  • 当将运算符作用到C风格字符串时，实际比较的是指针而非字符串本身。

  • 对于大多数应用来说，使用string要比C风格字符串更加安全高效。

  • 允许使用以空字符串结束的字符数组初始化string对象或者为对象赋值

  • 在string对象加法运算中允许使用以空字符结束的字符数组作为其中的一个运算对象。

  • 无法使用string对象来替换C风格字符串，例如不能使用string对象直接初始化指向字符的指针，使用c_str()函数可以完成转换，返回的指针类型为const char * 这样保证我们不会改变字符数组的内容。

  • 无法保证c_str()函数返回的数组一直有效，如果后续操作改变了string对象的值，可能让返回得到的C风格字符串失去效用。因此最好重新拷贝一份。

  • 可以用数组初始化vector对象，实现这一目的，只需要指明拷贝区域的首元素地址和尾后元素地址

int init_arr[] = {1, 3, 4, 5, 6}; vector ivec (begin(init_arr), end(init_arr));
