参数传递

  • 使用引用避免拷贝，拷贝大的类类型对象或者容器对象比较低效，甚至有的类类型不支持拷贝操作，因此函数只能通过引用形参访问该类型的对象。

  • 使用实参初始化形参的时候会忽略掉const，当形参有顶层const的时候，传给它常量或者非常量都是可以的。

void fcn(const int i) //调用fcn函数的时候既可以传入const int 也可传入int。

const int ci = 43; int k = ci; //当拷贝ci的时候，忽略了顶层的const

  • 可以用非常量初始化一个底层const对象， 但是反过来不行。

  • 普通引用不能被字面值初始化，也不能被常量初始化，const int & i = 常量 // 这样可以

因此函数不会改变的形参应该定义成常量引用，这样能把const对象，字面值当成实参初始化形参。

数组形参

数组有两个特殊性质：不允许拷贝数组，使用数组通常会将其转换成为指针，因为不能拷贝数组因此不能以值传递的方式使用数组参数，传递数组的时候。实际上传递的是指向数组首元素的指针。

管理指针形参的3种常用技术：

  1. 使用标记指定数组长度：要求数组本身包含一个结束标记。例如C风格字符串需要有结束标记。但是对于像int这种所有取值都合法的值就不太有效了。

  2. 使用标准库规范：传递指向数组的首元素和尾元素指针。

  3. 显示传递一个表示数组大小的形参。

  4. 数组引用形参： 型如下 void print( int (&arr)[10]) 形参为具有十个整数的整型数组的引用。
