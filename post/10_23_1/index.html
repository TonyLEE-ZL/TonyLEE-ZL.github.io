基础

  • 左值：当对象被用作左值的时候，用的对象在内存中的位置。 右值：当一个对象被用作右值的时候用的是对象的值。

  • 赋值运算符需要一个左值运算对象，得到结果仍然是一个左值。

  • 取地址符号作用与一个左值，返回一个指向对象的指针，这个指针是一个右值。

  • 内置类型和迭代器递增和递减运算符作用与左值对象，其前置版本所得的结果也是左值。

  • 内置解引用运算符，下标运算符 的求值结果为左值。

求值顺序：对于那些没有指定求值顺序的表达式来说如果表达式指向并修改了同一个对象，将会引发并产生未定义的错误。

1
2
3

int i = 0;
cout << i << "" << ++i
//编译器可能先求i的值在求i++的值或者另外的顺序，因此表达式行为不可知，产生错误

算术运算符

一元算数运算符优先级最高，一元运算符都是左结合律。按照从左到右顺序组合。

赋值 递增递减运算符

赋值运算符的左侧运算对象必须是一个可以修改的左值。结果类型为左值。如果赋值运算符左右两个运算对象类型不同，则右侧的运算对象类型转换成左侧类型。满足右结合律，

赋值运算符的优先级较低 切勿混淆使用相等运算符和赋值运算符。

前置和后置运算符：前置版本返回对象本身作为左值返回。 后置版本则是将对象的原始值的副本作为右值返回。

  • 建议 除非必须，否则不用递增运算符的后置版本。

在一条语句中混用解引用和递增运算符：如果我们想在一条符复合表达式中既将变量加减一又能使用原来的值 就可以用递增递减的后置版本了。 例子：

auto pbeg = v.begin();
while(pbeg != v.end && *pbeg >= 0)
    cout << *pbeg << endl;
    //由于后置递增符号返回的初始值的副本因此答应的值为原始值，
    //如果用前置，不能输出第一个元素，如果没有非负元素解引用一个不存在的元素造成巨大的错误。

大多数的运算符没有规定运算对象的求值顺序，然而如果一条子表达式改变了某个运算对象的值，另外一个子表达式使用了该值，运算对象的求值顺序就很关键了。

while(beg != s.end() && !isspace(*beg))
    *beg = toupper(*beg++)
    //两边都用beg且右边改变了beg的值 由于对象的求值顺序不一致因此会造成未定义的问题

    *beg = toupper(*beg); //如果先求左边的值
    *(beg + 1) = toupper(*beg); //如果先求右边的值
