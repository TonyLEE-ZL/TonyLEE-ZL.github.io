位运算符

位运算符作用于整数类型的运算对象。并把运算对象看成式二进制位的集合。

移位运算符

<< 左移运算符，bits << 8 首先令左侧运算内容向左侧移动8位，移位之后可能提升。例如 bits为char类型，向左移动后变为int类型。左移之后的右侧插入0的二进制位。如果移动后左边超出边界，超出的部分将丢弃掉，

>> 右移运算符运算对象一定不能为负数，而且结果必须小于结果的位数。右移运算符的行为依赖于运算对象的类型。如果该类型是无符号型的在左侧插入0的二进制位。如果运算对象是带符号型的，在左侧插入符号位的副本或者0的二进制位。

sizeof运算符

返回一条表达式或者一个类型名字所占的空间。sizeof不会求运算对象的值，所以即使是一个无效指针也能求出它的大小。

  • 对char或者类型为char的表达式执行sizeof结果为1.
  • 对引用类型执行sizeof运算得到被引用的对象所占的空间大小
  • 对指针执行sizeof运算得到指针本身所占空间大小。
  • 对解引用指针执行sizeof运算得到指针所指向的对象空间所占的大小，指针不需要有效。
  • 对数组执行sizeo运算得到整个数组所占空间大小。sizeof不会将数组转化成为指针。
  • 对string和vector执行sizeof返回该类型固定部分的大小，不会计算对象中的元素占用了多少空间。

类型转换

  • 整型提升：对于bool char unsigned char short 等类型来说，只要他们的值在int里，它们就会提升为int类型否则提升为unsigned int类型。 较大char类型，(wchar_t, char16_t,)提升成为int long，long long 等类型中的最小的一个，前提是转换后能容纳原类型的所有可能的值。

其他隐式类型转换

  • 指针转换：转换成布尔类型，存在从算术类型或指针类型向布尔类型自动转换的机制。

  • 转换成常量 允许将指向非常量类型的指针转换成指向相应常量类型的指针和引用 但是不允许将const转换成非常量。

  • 类类型定义的转换

显示转换

  • static_cast 任何具有明确意义的类型转换，只要不包含底层const，都可以使用static_cast.对于编译器无法自动执行的类型转换也非常有用。例如可以使用其找回void*指针中的值。

void *p = &d; //任何非常量对象的地址都能存入void*

double *dp = static_cast(p);// 可以将p转换到原来的类型，确保指针的值不变。

  • const_cast 对于常量对象转换成非常量对象的行为，称其为去const性质。 const_cast 只能改变运算对象的底层const 但是不能通过生成的对象改变指向的值。const_cast用于函数重载上下文当中。先用该值但是函数参数为非const 且不改变该对象的值，因此使用该函数去掉底层的const。

const char * pc;

char *p = const_cast(pc); // 可以去掉底层const 但是通过p写入值是未定义的行为。

  • reinterpret_cast 可以进行不同类型的指针转换。
